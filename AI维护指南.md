# AI 下单助手维护指南

本文档给后续接手本项目的 AI 或开发者阅读。  
目标是防止后续修改时把系统重新改回“依赖 AI 猜业务字段”的不稳定状态。

## 1. 项目一句话说明

这是一个订单录入辅助系统。

用户输入一段自然语言订单后，系统需要：

- 拆出收货人、电话、地址。
- 拆出货品、数量、价格。
- 根据固定表格匹配标准货品。
- 根据固定规则判断业务员、客户账号、付款方式、收款账户、快递。
- 生成可导出的订单表和货品表。

本项目的核心原则是：

```text
AI 只负责切割订单原文。
最终业务字段必须由本地规则、Excel 表格、JSON 配置决定。
```

## 2. 最重要的维护原则

后续修改必须遵守：

```text
不要让 AI 决定最终货品。
不要让 AI 决定最终快递。
不要让 AI 决定最终业务员。
不要让 AI 决定最终收款账户。
不要用模糊匹配自动选货品。
查不到就待确认，不要猜。
```

如果某个字段识别不准，优先补本地规则、表格或配置，不要优先改 AI 提示词。

## 3. 当前架构分工

### AI 负责

AI 只负责粗切订单原文：

- 收货人
- 手机号
- 地址
- 省份
- 货品原文片段
- 数量
- 备注原文

货品字段应使用：

```json
{
  "raw_name": "订单原文里的货品文字",
  "qty": 1,
  "price": 0
}
```

AI 不应该输出或决定标准货品名。

### 本地规则负责

以下字段必须由本地规则决定：

- 标准货品
- 条码
- 规格编号
- 货品编号
- 业务员
- 客户账号
- 付款方式
- 收款账户
- 快递公司
- 客服备注
- 导出字段

## 4. 货品匹配规则

货品匹配是本项目最重要的规则，不能随意改。

### 4.1 收银机和一体称

收银机和一体称只查：

```text
order-helper-backend/data/总库存-组合装明细.xlsx
```

唯一匹配字段：

```text
货品名称
```

规则：

- 订单中的机器名称必须精确匹配该表 `货品名称`。
- 匹配成功后才自动确认。
- 不能用别名。
- 不能用模糊匹配。
- 不能用包含匹配。

### 4.2 普通货品

普通货品只查：

```text
order-helper-backend/data/总库存.xlsx
```

唯一匹配字段：

```text
规格编号
```

规则：

- 订单中必须出现 `规格编号`。
- 系统只用 `规格编号` 精确匹配普通货品。
- 不用 `货品名称` 自动匹配普通货品。
- 不用货品编号、条码、别名自动推断普通货品。
- 不做模糊匹配。

示例：

```text
XP-58TS        -> 不自动匹配
XP-58TS(USB)  -> 匹配到 XP-58TS
```

### 4.3 括号归一化

规格编号匹配允许轻量归一化：

- 中文括号和英文括号视为同一种括号。
- 空格差异忽略。

示例：

```text
订单：S-H80（USB+网口+串口）
表格：S-H80(USB+网口+串口）
结果：可以匹配
```

这仍然是精确匹配，不是模糊匹配。

### 4.4 未匹配处理

如果货品查不到：

- 后端设置 `needsReview = true`。
- 前端显示 `[待确认]`。
- 快递不自动判定。
- 用户需要手动从全库搜索选择。

不要自动选一个相似货品。

## 5. 价格识别规则

价格优先从订单原文本地解析。

支持公式：

```text
90*2=180元
95*3=285元
3*95=285元
90+210=300
```

规则：

- 乘法公式用于识别单个货品的单价。
- 加法公式按货品顺序拆分多个货品价格。
- AI 没给价格时，本地规则也要尽量解析。
- 如果价格公式无法确定，不要乱填。

示例：

```text
JY-335C黑*1+S-H80（USB+网口+串口）*1 90+210=300
```

应识别为：

```text
JY-335C：90
佳博S-H80：210
```

## 6. 快递规则

快递不由 AI 决定。

快递判断来源：

```text
order-helper-backend/data/快递表格.xls
```

快递判断依赖：

- 收货省份
- 已确认货品
- 货品重量
- 是否收银机
- 是否一体称
- 用户是否手动指定快递

### 6.1 快递表字段映射

`快递表格.xls` 中打印机重量段列需要映射为：

| Excel 原列 | 程序字段 |
| --- | --- |
| 打印机 | 打印机_小 |
| Unnamed: 2 | 打印机_中 |
| Unnamed: 3 | 打印机_大 |
| 收银机 | 收银机 |
| 一体称 | 一体称 |

不要删除该映射。

### 6.2 未确认货品时

只要订单中存在 `needsReview = true` 的货品：

```text
快递 = 待确认
```

不要在货品未确认时自动判定快递。

### 6.3 手动指定快递

如果订单原文明确写：

```text
发顺丰
走德邦
指定圆通
```

手动指定优先。

关键词在：

```text
order-helper-backend/core/express.py
```

变量：

```python
manual_keywords
```

新增快递指定词时，只改这里。

## 7. 业务员规则

业务员不由 AI 决定。

优先级：

1. 订单原文括号代码，例如 `（L）`、`(TX)`。
2. 客户档案。
3. 默认业务员。

映射位置：

```text
order-helper-backend/data/配置规则.json
```

字段：

```json
"业务员映射": {}
```

如果新增业务员，改这里。

## 8. 客户账号规则

客户账号优先从原文识别：

```text
系统录
系统录入
系统录制
系统记录
```

规则：

- `系统录` 后面的内容作为客户账号。
- 该规则优先级高于 AI。
- AI 不应覆盖该字段。

## 9. 付款方式和收款账户规则

付款方式和收款账户不由 AI 决定。

规则位置：

```text
order-helper-backend/data/配置规则.json
```

字段：

```json
"收款账户映射": {}
```

付款方式：

- 出现 `已付`、`转账`、`已转` -> `银行收款`
- 出现 `未付`、`欠款` -> `欠款计应收`
- 默认 -> `欠款计应收`

新增账户示例：

```json
"已付新账户": "新账户名称"
```

不要通过 AI 判断收款账户。

## 10. 客服备注规则

客服备注由系统合成，不由 AI 自由生成。

格式：

```text
货品原始摘要 + 原文备注内容
```

示例：

```text
JY-335C黑*1 咸阳市泾阳县崇文镇文定天下店
```

如果有两个以上货品，货品摘要之间使用 `+` 连接：

```text
JY-335C黑*1+S-H80（USB+网口+串口）*1 咸阳市泾阳县崇文镇文定天下店
```

备注逻辑主要在：

```text
order-helper-backend/core/processor.py
```

## 11. 前端规则

前端主要负责展示和人工修正。

前端必须保留：

- 未匹配货品的 `[待确认]` 提示。
- 全库搜索功能。
- 人工选择货品后清除 `needsReview`。
- 价格、数量修改后重算合计。
- 结算方式和收款账户联动。

不要在前端偷偷做模糊自动匹配。

## 12. 后续新增规则怎么做

优先顺序：

```text
先改表格或 JSON 配置。
再改确定性规则函数。
最后才考虑 AI 提示词。
```

常见新增位置：

| 要新增什么 | 修改哪里 |
| --- | --- |
| 新业务员 | `配置规则.json` -> `业务员映射` |
| 新收款账户 | `配置规则.json` -> `收款账户映射` |
| 新快递下拉项 | `配置规则.json` -> `快递选项` |
| 快递省份重量规则 | `快递表格.xls` |
| 原文指定快递关键词 | `core/express.py` -> `manual_keywords` |
| 新普通货品 | `总库存.xlsx` -> `规格编号` |
| 新收银机/一体称 | `总库存-组合装明细.xlsx` -> `货品名称` |

## 13. 修改前必须检查的问题

后续 AI 或开发者改代码前，必须先回答：

1. 这个字段是不是死规则？
2. 这个字段是不是已经在 Excel 或 JSON 里？
3. 能不能通过精确查表解决？
4. 查不到时是否应该待确认？
5. 改动会不会让 AI 重新参与最终判断？

如果答案会导致 AI 猜最终字段，应停止这种修改。

## 14. 禁止改法

不要做以下修改：

- 恢复货品模糊匹配。
- 恢复货品包含匹配。
- 让 AI 输出标准货品名并直接采用。
- 让 AI 输出最终快递并直接采用。
- 让 AI 输出最终收款账户并直接采用。
- 货品未确认时仍自动判快递。
- 把普通货品改成按 `货品名称` 自动匹配。

这些都会让系统重新变得不稳定。

## 15. 推荐测试样例

每次修改后至少测试：

```text
XP-58TS
```

预期：不自动匹配。

```text
XP-58TS(USB)
```

预期：匹配普通货品。

```text
S-H80（USB+网口+串口）*1
```

预期：匹配 `佳博S-H80`。

```text
JY-335C黑*1+S-H80（USB+网口+串口）*1 90+210=300
```

预期：

- 识别两个货品。
- 数量都是 1。
- 单价分别为 90 和 210。
- 合计 300。

```text
山东 XP-80U(USB+网口)*3 95*3=285
```

预期：

- 单价 95。
- 总重按库存计算。
- 山东 2-5kg 按快递表走 `圆通（渠道）`。
