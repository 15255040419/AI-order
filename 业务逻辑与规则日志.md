# 订单助手系统 - 业务逻辑与改进规则日志

本文档记录了系统在开发过程中针对特定业务场景所实施的所有定制化逻辑与 AI 识别规则，以便后续维护与升级。

---

## 1. AI 智能解析规则 (Prompt Logic)

### 1.1 业务员识别
- **规则**：业务员必须通过括号内的字母代码识别（如 `(W)` 或 `（L）`）。
- **优化**：严禁 AI 根据上下文猜测业务员。若文本中无括号包裹的字母，`salesman_code` 必须返回空。支持双括号容错（如 `（W））`）。

### 1.2 客户账号提取 (重点)
- **规则**：查找到“**系统录**”或“**录**”关键字时，其后的名字或代号必须提取为 `customer_account`。
- **防止误放**：严禁 AI 将该信息错放进 `extra_note`（备注）字段。

### 1.3 货品与规格解析
- **复杂名处理**：针对名称本身含星号的货品（如 `57*50*10卷纸`），要求 AI 将 `57*50` 识别为名称，`10` 识别为数量，不得将数量后缀带入货品名。
- **价格算式**：支持解析 `275+10*0.5=280` 这种复合算式。

## 2. 后端核心逻辑 (Backend Processing)

### 2.1 客户匹配 (Sanitization & Matching)
- **正则兜底**：如果 AI 没抓到账号，后端会通过正则直接锁死关键字“系统录”之后的文字进行强行提取。
- **档案反查**：支持通过抓取到的名字或联系电话，自动在《客户档案.xlsx》中检索正式的“客户账号”。
- **符号转义**：在查找客户时，强制禁用正则模式（`regex=False`），防止手机号中的 `+` 号导致程序崩溃。

### 2.2 货品模糊匹配 (阶梯式优先级)
- **第一优先级：规格编号**。后端会首先尝试在“规格编号”列中进行双向包含匹配。一旦命中该列，系统即认为找到了最精准的货品，不再继续扫描其他列。
- **第二优先级：货品名称与规格**。若规格编号未命中，系统会并发扫描“货品名称”和“规格”两列，寻找最贴切的候选。
- **长度比对策略**：在同级权重匹配中，依然采用“长度绝对差最小”原则，确保如输入 `57*50` 能精准命中库中的 `57*50纸`。
- **元数据补全**：匹配成功后，自动补齐条码、货品编号等所有关联元数据。

### 2.3 自动归一化
- **收款账户**：根据《配置规则.json》中的映射表，自动将“已付农商成”、“已付公账”等多样化表述统一为系统规范名称。

### 2.4 数据安全性与稳定性 (Stability Fixes)
- **特殊符号脱敏**：针对手机号中的 `+` 等正则元字符，在 pandas 查询时强制使用 `regex=False`，防止特殊输入导致后端崩溃（`re.PatternError`）。
- **NaN 值自动清洗**：针对 Excel 中常见的空白单元格，后端增加了递归清洗器（`sanitize_data`），在返回 JSON 前将所有 `NaN` 转换为 `None` (null)，彻底解决了由于 JSON 格式不规范导致的网页显示中断问题。
- **跨域与通信优化**：统一使用全局 `CORS(app)` 配置，移除了冗余且易报错的单个接口装饰器，确保本地 HTML 文件与 Flask 后端的高性能、稳定通信。

### 2.5 客户账号查找强化 (Account Matching v2)
- **关键字优先**：后端逻辑现在强制优先扫描原文中的“系统录”关键字。即便是 AI 抓取到的名字带了括号（如 `（W）`），后端也会在反查档案前自动将其剥离，确保匹配纯净。
- **防止误包含**：修正了如果 AI 猜测的账号包含杂质时导致档案匹配失败的问题。

### 2.6 货品名防污染 (Product Anti-Pollution)
- **星号冲突处理**：针对 `57*50*10` 这种特殊规格，明确了 AI 必须将最后一个 `*10` 视为数量，前面的 `57*50` 视为名称。通过在 Prompt 中增加“阴阳对比”示例和“严禁污染”指令，大幅提升了规格货品的识别率。

### 2.9 代码稳定性红线 (Technical Anti-Pollution)
- **禁用多余装饰器**：严禁在接口上使用 `@cross_origin()`。系统中已在全局层面通过 `CORS(app)` 实现了跨域管理。重复添加该装饰器会导致程序因缺少导入而“闪退”（打不开）。
- **统一导入规范**：所有核心库（如 `math`, `re`, `json`）必须统一在文件顶部导入，严禁在函数内部零星导入，以确保系统启动的鲁棒性。

## 3. 前端交互与 UI 逻辑 (UE/UX)

### 3.1 微信对话流式交互 (Chat Flow)
- **布局设计**：采用左侧气泡、右侧卡片的对齐布局。左侧气泡模仿微信绿背景，保留原始粘贴文本。
- **高度同步**：左右两侧内容在垂直高度上保持一致，方便人工核对。

### 3.2 录入操作优化
- **键盘支持**：输入框支持 `Enter` 直接发送识别，`Shift + Enter` 实现换行。
- **粘贴自触发**：检测到粘贴行为后 50ms 自动触发解析，无需手动点击。
- **异步解析队列**：支持多订单连续快速粘贴，通过唯一索引位确保解析返回的数据不会覆盖错位。

### 3.3 数据展示
- **详情透明化**：货品明细表格支持点击展开，显示条码、编号、规格、数量及单价。
- **错误可视化**：若解析失败，卡片标题变红并显示具体的 HTTP 状态码或网络错误描述。

---

## 4. Excel 导出规范
- **模板对齐**：导出后的 Excel 严格遵循《订单分表模版.xlsx》的列顺序和工作表命名（“订单”+“货品”）。
- **导入编号关联**：多订单导出时，通过“导入编号”自动关联两个 Sheet 之间的数据。

## 5. 视觉工程 (Figma Pro Upgrade)

### 5.1 1:1 原型还原
- **设计迁移**：通过引入 Tailwind CSS 和 Lucide 图标库，将 Figma 专业版设计完整移植到了单 HTML 文件中。
- **现代化组件**：实现了磨砂玻璃导航栏、动态光影渐变标题、以及更丝滑的卡片悬浮动效。

### 5.2 极致密度提升
- **密集栅格布局**：升级为 3 列密集栅格布局（收货/合计/账号三位一体），结合扁平化的明细表单，确保单屏处理效率比初代提升 40% 以上。
- **状态感知**：增加了解析中的 Loading 骨架屏和状态闪烁微动画，提升了系统的 呼吸感。

### 5.3 生产力加固
- **候选人优选**：前端现已完美支持从 Top 10 相似度候选列表中进行快速修正。
- **零成本部署**：通过云端 CDN 方案，不仅保留了 Figma 的专业感，且依然支持双击即用，延续了单文件部署的极低维护成本。
