# 业务逻辑与规则日志 (2026-04-18 升级版)

## 1. 核心技术沉淀

### 1.1 高精度货品匹配引擎 (find_product_match)
*   **双向包含逻辑**：修正了匹配方向。不仅支持“库名包含解析名”，也支持“解析名包含库名”。确保如 `T80Q` 能稳稳匹配到 `XP-T80Q`。
*   **多表头兼容提取**：后端 `app.py` 实现了 `get_field` 兼容函数。无论 Excel 表头叫“规格编号”、“货品代码”还是“编码”，系统均能自动识别并提取条码、规格等关键元数据。
*   **数量后缀剥离**：通过正则 `re.sub(r'[*xX]\d+$', '', name)` 自动剔除型号末尾的 `*2`、`x2` 等数量标识，防止干扰型号匹配。

### 1.2 客户购买习惯记忆 (Customer History)
*   **结构化存储**：`customer_history.json` 升级为全量对象结构。不再只存名字，而是存储包含 `spec_no`、`barcode`、`spec` 的完整“货品身份证”。
*   **一次匹配，终身受益**：AI 首次解析并手动修正后，该客户的特定购买习惯将被锁死在 JSON 档案中，下次相同关键词将 100% 精确回填所有属性。

### 1.3 前端极速处理 UI (Order Assistant v2.0)
*   **紧凑布局**：压缩卡片尺寸，增加信息密度，实现单屏展示更多订单。
*   **账号自由输入**：将客户账号由下拉框改为高亮输入框，支持手动即时修改。
*   **原文追溯**：在卡片底部回显订单原文，确保“有据可查”，极大降低了人工核对成本。
*   **手动同步逻辑**：在前端 `updateProduct` 中加入了异步同步机制。手动切换货品名称时，系统会自动从本地缓存或后端 API 补全该货品的条码和规格。

## 2. 系统硬规则记录

*   **结算方式**：默认优先识别“已付”字样。若未指定或带有“系统录”等词汇，默认映射为“欠款计应收”或指定账户。
*   **快递逻辑**：优先匹配 `配置规则.json` 中的“地址关键词 -> 快递”映射关系。其次参考历史档案。
*   **灵动视觉**：首页 Mascot 小人实现“神形合一”。嘴巴与眼睛同步监听鼠标位移，提升用户交互时的愉悦感。
