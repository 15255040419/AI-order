# AI 下单助手 (Order Assistant) - 智能 workbench

这是一个基于 AI 驱动的自动化订单处理系统，旨在通过 AI 语义解析与硬性业务规则相结合，实现订单的极速录入、货品校验及快递自动分拣。

## 📁 核心架构说明 (Backend Core)

系统采用模块化设计，业务逻辑高度封装在 `core/` 文件夹中，各模块职责如下：

| 模块名称 | 核心职责 | 业务关联 |
| :--- | :--- | :--- |
| **ai_parser.py** | **大脑** | 负责对接 AI 接口（智谱），进行初步的语义提取。 |
| **loader.py** | **搬运工** | 负载从 Excel 和 JSON 配置文件中动态装载库存、客户、快递规则等。 |
| **matcher.py** | **火眼金睛** | 货品二级校验引擎，负责将 AI 提取的模糊货名与库存表进行 100% 对齐。 |
| **processor.py** | **组装车间** | **灵魂模块**。负责强力正则锁（业务员代码、系统录账号、备注合成、金额兜底）。 |
| **express.py** | **快递员** | 根据包裹重量、产品品类及收货省份，实现精准分拣与地址杂质剔除。 |
| **customer.py** | **老客秘书** | 负责同步客户档案，回填业务员偏好及历史交易规则。 |
| **address.py** | **地图** | 净化地址字符串，精准识别省份信息，为快递判定提供依据。 |
| **mapping.py** | **翻译官** | 管理所有同义词及业务代码映射（如收款账户、业务员代号）。 |

## 🚀 业务逻辑速览

1. **强力正则锁**：系统不迷信 AI，通过硬性正则表达式优先识别括号内的业务员代号 `(L)` 及 `系统录:` 后的客户账号。
2. **备注强力合成**：客服备注字段固定由 `[货品摘要] + [原文中备注字样后的文字]` 组成，确保仓库可见性。
3. **分拣防干扰**：在判定快递前，系统会自动从地址中扣除干扰项，杜绝如“圆通北路”导致的发错快递。
4. **数据驱动**：所有规则均通过 `data/配置规则.json` 管理，修改配置立即可生效，无需修改代码。

## 🛠️ 如何启动

1. 进入后端目录：`cd order-helper-backend`
2. 安装依赖：`pip install -r requirements.txt`
3. 运行服务：`python app.py`
4. 打开前端：直接双击根目录下的 `order-assistant-full.html` 即可。
