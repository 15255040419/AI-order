# 吉客云 - 智能订单处理助理 (AI Order Assistant)

一个高效、美观且智能的订单处理工作台，旨在将碎片化的原始订单文本快速转化为规范化的 Excel 数据。

## 🚀 核心功能

*   **混合架构解析 (AI + Local)**：采用“AI提取非结构化信息 + 本地正则锁定关键字段”的双引擎模式。AI 负责解析地址、姓名；本地逻辑负责锁定业务员代码、客户账号、系统录等固定标志，兼顾灵活性与准确性。
*   **SKU 智能匹配 (Local Logic)**：内置高性能模糊评分算法（基于 `difflib`），在本地毫秒级完成订单货品与《总库存.xlsx》的对齐，无需将海量库存表上传云端。
*   **业务逻辑联动**：实现“支付状态-收款账户”自动对齐。例如选定“欠款”则自动清空账户，选定账户则自动切换“银行收款”，杜绝财务对账错误。
*   **快递自动识别**：全自动策略路由。基于省份、重量及特定关键词（如“顺丰”、“工厂代发”）在本地查表匹配最佳快递方案。
*   **Windows 环境深度优化**：针对 Windows 控制台编码（GBK）进行了专项适配，支持 UTF-8 启动模式，解决中文字符与 Emoji 导致的崩溃问题。
*   **高密度 UI 设计**：专为批量处理优化，支持实时编辑与动态反馈，大幅提升审单效率。
*   **一键数据导出**：完美衔接吉客云等 ERP 业务环节，支持 Excel 分表导出。

## 🛠️ 技术栈

*   **后端**：Python (Flask)
*   **前端**：HTML5, JavaScript (ES6+), Tailwind CSS (Vanilla)
*   **数据驱动**：Pandas, Openpyxl, XLSX.js

## 📁 项目结构

*   `order-assistant-full.html`：全功能前端工作台。
*   `order-helper-backend/`：Flask 后端服务，包含逻辑处理与数据匹配。
*   `order-helper-backend/data/`：存放库存、客户及规则配置文件的核心目录。
*   `业务逻辑与规则日志.md`：详细记录了版本迭代与核心技术逻辑。

## 📝 使用指南

1.  启动后端服务：运行 `python app.py`。
2.  打开前端页面：在浏览器中访问 `order-assistant-full.html`。
3.  输入订单原文：将复杂的订单消息复制到输入框，点击发送即可自动生成订单报表。
