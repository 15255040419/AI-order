# AI 下单助手

这是一个用于线下订单录入的智能辅助系统。

用户粘贴一段订单原文后，系统会自动拆分收货信息、货品、价格、付款方式、客户账号、业务员、快递等字段，并生成可导出的订单表和货品表。

## 重要原则

本项目当前已经从“让 AI 猜结果”调整为“AI 切原文 + 本地规则查表”。

核心原则：

```text
AI 只负责切割订单原文。
最终业务字段必须由本地规则、Excel 表格、JSON 配置决定。
查不到就待确认，不自动乱猜。
```

后续 AI 或开发者修改本项目之前，必须先阅读：

```text
AI维护指南.md
```

禁止恢复以下做法：

- 让 AI 决定最终标准货品。
- 让 AI 决定最终快递。
- 让 AI 决定最终收款账户。
- 让 AI 决定最终业务员。
- 用模糊匹配自动选择货品。
- 货品未确认时仍自动判定快递。

## 当前业务逻辑

### 订单解析

AI 只用于粗拆订单原文，例如：

- 收货人
- 手机号
- 收货地址
- 省份
- 货品原文片段
- 数量
- 备注原文

AI 不负责最终业务判断。

### 货品匹配

货品匹配完全按表格规则执行。

收银机和一体称：

```text
order-helper-backend/data/总库存-组合装明细.xlsx
```

只查字段：

```text
货品名称
```

普通货品：

```text
order-helper-backend/data/总库存.xlsx
```

只查字段：

```text
规格编号
```

示例：

```text
XP-58TS        -> 不自动匹配
XP-58TS(USB)  -> 匹配普通货品 XP-58TS
```

如果货品无法精确匹配，系统会标记为待确认，前端需要人工选择。

### 价格识别

系统会从订单原文中识别确定性价格公式。

支持示例：

```text
90*2=180元
95*3=285元
3*95=285元
90+210=300
```

例如：

```text
JY-335C黑*1+S-H80（USB+网口+串口）*1 90+210=300
```

会识别为：

```text
JY-335C：90
佳博S-H80：210
```

### 快递判断

快递不由 AI 决定。

快递规则主要来自：

```text
order-helper-backend/data/快递表格.xls
```

判断依据：

- 省份
- 已确认货品
- 货品重量
- 是否收银机
- 是否一体称
- 用户是否手动指定快递

如果存在未确认货品，快递直接显示为：

```text
待确认
```

### 业务员

业务员优先从订单原文括号代码识别，例如：

```text
（L）
(TX)
```

映射配置在：

```text
order-helper-backend/data/配置规则.json
```

字段：

```json
"业务员映射": {}
```

### 客户账号

客户账号优先识别订单原文中的：

```text
系统录
系统录入
系统录制
系统记录
```

这些关键词后面的内容会作为客户账号。

### 付款方式和收款账户

付款方式和收款账户由本地规则判断。

配置位置：

```text
order-helper-backend/data/配置规则.json
```

字段：

```json
"收款账户映射": {}
```

付款方式规则：

- 出现 `已付`、`转账`、`已转` -> `银行收款`
- 出现 `未付`、`欠款` -> `欠款计应收`
- 默认 -> `欠款计应收`

### 客服备注

客服备注由系统固定合成：

```text
货品原始摘要 + 原文备注内容
```

例如：

```text
JY-335C黑*1 咸阳市泾阳县崇文镇文定天下店
```

## 数据文件说明

核心数据文件都在：

```text
order-helper-backend/data/
```

| 文件 | 作用 |
| --- | --- |
| `总库存.xlsx` | 普通货品数据，普通货品按 `规格编号` 匹配 |
| `总库存-组合装明细.xlsx` | 收银机、一体称数据，按 `货品名称` 匹配 |
| `快递表格.xls` | 快递省份、重量段、收银机、一体称规则 |
| `客户档案.xlsx` | 客户信息、手机号、业务员、客户账号 |
| `配置规则.json` | 业务员、收款账户、快递选项等固定配置 |
| `customer_history.json` | 老客户购买习惯预留数据 |

## 项目结构

```text

  core/
    ai_parser.py         AI 订单原文切割

    express.py           快递判断

    配置规则.json

    总库存-组合装明细.xlsx

```

## 启动方式

进入后端目录：

```bash
cd order-helper-backend
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动服务：

```bash
python app.py
```

打开前端：

```text
order-assistant-full.html
```

或访问后端首页：

```text
http://127.0.0.1:5005
```

## 后续新增规则

优先改表格或 JSON 配置，不要优先改 AI。

| 要新增什么 | 修改哪里 |
| --- | --- |
| 新业务员 | `配置规则.json` -> `业务员映射` |
| 新收款账户 | `配置规则.json` -> `收款账户映射` |
| 新快递下拉项 | `配置规则.json` -> `快递选项` |
| 快递省份重量规则 | `快递表格.xls` |
| 原文指定快递关键词 | `core/express.py` -> `manual_keywords` |
| 新普通货品 | `总库存.xlsx` -> `规格编号` |
| 新收银机/一体称 | `总库存-组合装明细.xlsx` -> `货品名称` |

## 修改前检查清单

后续修改前必须确认：

- 这个字段是不是死规则？
- 这个字段是不是已经在 Excel 或 JSON 里？
- 能不能通过精确查表解决？
- 查不到时是不是应该待确认？
- 改动会不会让 AI 重新参与最终判断？

如果改动会让 AI 猜最终业务字段，应停止这种修改。

## 推荐测试样例

每次修改后建议至少测试：

```text
XP-58TS
```

预期：不自动匹配。

```text
XP-58TS(USB)
```

预期：匹配普通货品。

```text
S-H80（USB+网口+串口）*1
```

预期：匹配 `佳博S-H80`。

```text
JY-335C黑*1+S-H80（USB+网口+串口）*1 90+210=300
```

预期：

- 识别两个货品。
- 数量都是 1。
- 单价分别为 90 和 210。
- 合计 300。
